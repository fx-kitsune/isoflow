# Đóng góp

### Chiến lược phân nhánh:

Các nhánh (branches) được đặt tên theo quy ước sau:

- `feature/` cho các triển khai tính năng mới
- `fix/` cho các bản sửa lỗi mã nguồn bị hỏng / lỗi build / sửa lỗi bug
- `chore/` cho các thay đổi mã nguồn không gây lỗi hỏng hóc & không phải sửa lỗi, chẳng hạn như dọn dẹp mã (linting), định dạng (formatting), v.v.

### Chiến lược Commit / PR (Pull Request):

- Các commit phải được gộp lại (squashed) trước khi merge
- Mỗi PR chỉ nên nhắm vào một vấn đề duy nhất để giữ cho lịch sử commit gọn gàng và dễ theo dõi

### Triển khai lên NPM

CI (Tích hợp liên tục) sẽ tự động kích hoạt đối với bất kỳ thẻ (tag) nào được đẩy lên nhánh `main`. Nó sẽ xây dựng và triển khai ứng dụng lên NPM.
Để triển khai:

1. Tăng phiên bản bằng cách sử dụng `npm version patch` hoặc tương tự
2. `git push && git push --tags`
