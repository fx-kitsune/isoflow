# `initialData`

Đối tượng `initialData` chứa các thuộc tính sau:

| Tên | Kiểu | Mặc định |
| --- | --- | --- |
| `version` | `string \| undefined` | `undefined` |
| `title` | `string \| undefined` | `undefined` |
| `description` | `string \| undefined` | `undefined` |
| `icons` | [`Icon[]`](#icon) | `[]` |
| `colors` | [`Color[]`](#color) | `[]` |
| `items` | [`Item[]`](#item) | `[]` |
| `views` | [`View[]`](#view) | `[]` |
| `fitToScreen` | `boolean` | `false` |

## Ví dụ về payload
Bạn có thể tìm thấy một payload ví dụ tại đây trên [CodeSandbox](https://codesandbox.io/p/sandbox/github/markmanx/isoflow/tree/main).

## `Icon`

```js
{
  id: string;
  name: string;
  url: string;
  collection?: string;
  isIsometric?: boolean;
}
```

**Lưu ý về Biểu tượng (Icons):**
- `collection` là một thuộc tính tùy chọn có thể được sử dụng để nhóm các biểu tượng lại với nhau trong bộ chọn biểu tượng. Tất cả các biểu tượng có cùng `collection` sẽ được nhóm lại dưới tên của collection đó.
- Nếu bạn đang sử dụng một hình ảnh biểu tượng không thuộc dạng isometric, bạn có thể thiết lập `isIsometric` thành `false` để hiển thị nó với một góc nhìn isometric và giúp nó hòa nhập vào bối cảnh tốt hơn.

## `Color`

```js
{
  id: string;
  value: string;
}
```

**Lưu ý về Màu sắc (Colors):**
Thuộc tính `value` chấp nhận mọi giá trị màu sắc trong một định dạng hợp lệ của CSS (ví dụ: một giá trị hex như `#000000`, một giá trị chuỗi ví dụ như `black` hoặc một giá trị rgba như `rgba(0, 0, 0, 1)`).

## `Item`

```js
{
  id: string;
  name: string;
  description?: string;
  icon: string;
}
```

**Lưu ý về Mục (Items):**
- Thuộc tính `description` có thể chấp nhận định dạng markdown.
- Thuộc tính `icon` chấp nhận `id` của một `Icon`.

## `View`

```js
{
  id: string;
  name: string;
  description?: string;
  items: ViewItem[];
  rectangles?: Rectangle[];
  connectors?: Connector[];
  textBoxes?: TextBox[];
}
```

**Lưu ý về Giao diện (Views):**
- Thuộc tính `description` có thể chấp nhận định dạng markdown.

## `ViewItem`

```js
{
  id: string;
  labelHeight?: number;
  tile: {
    x: number;
    y: number;
  }
}
```

**Lưu ý về Mục của Giao diện (ViewItems):**
- Thuộc tính `id` chấp nhận `id` của một `Item` (được định nghĩa ở cấp gốc).

## `Connector`

```js
{
  id: string;
  description?: string;
  color?: string;
  width?: number;
  style?: 'SOLID' | 'DOTTED' | 'DASHED';
  anchors: ConnectorAnchor[];
}
```

**Lưu ý về Các đường kết nối (Connectors):**
- Thuộc tính `color` chấp nhận một `id` của một `Color` (được định nghĩa ở cấp gốc).
- Một đường kết nối cần tối thiểu 2 điểm neo (anchors) để xác định điểm bắt đầu và kết thúc của nó. Nếu bạn muốn kiểm soát nhiều hơn đường đi của đường kết nối, bạn có thể chỉ định các điểm neo bổ sung mà đường kết nối sẽ đi qua.

## `ConnectorAnchor`

```js
id: string;
ref: 
  | {
      tile: {
        x: number;
        y: number;
      }
    }
  | {
      item: string;
    }
```

**Lưu ý về Các điểm neo đường kết nối (ConnectorAnchors)**
- Các điểm neo đường kết nối có thể tham chiếu đến một `tile` hoặc một `item`. Nếu tham chiếu đến một `item`, điểm neo sẽ có tính động và sẽ được gắn liền với vị trí của mục đó.
- Khi neo một đường kết nối vào một `item`, bạn phải chỉ định `id` của mục đang được tham chiếu.

## `Rectangle`

```js
{
  id: string;
  color?: string;
  from: {
    x: number;
    y: number;
  };
  to: {
    x: number;
    y: number;
  };
}
```

## `TextBox`

```js
{
  id: string;
  tile: {
    x: number;
    y: number;
  };
  content: string;
  fontSize?: number;
  orientation?: 'X' | 'Y';
}
```

## Xác thực (Validation)
Đối tượng `initialData` được xác thực trước khi Isoflow hiển thị bối cảnh, và sẽ có lỗi ném ra nếu phát hiện dữ liệu không hợp lệ.

Dưới đây là một số ví dụ về các lỗi thường gặp:
- Một `ConnectorAnchor` tham chiếu đến một `Item` không tồn tại.
- Một `Item` tham chiếu đến một `Icon` không tồn tại.
- Một `Rectangle` có thuộc tính `from` nhưng lại thiếu thuộc tính `to`.
- Một `Connector` có ít hơn 2 điểm neo.
