# Isopacks

**Isopacks** là các mô-đun bổ sung cho Isoflow, chứa các biểu tượng và các tài nguyên khác. Bạn có thể dễ dàng tự xây dựng isopacks từ đầu hoặc tạo và tải isopacks của riêng bạn.

### Các Isopacks cho Đám mây & Mạng

Chúng có sẵn như một dự án được duy trì riêng biệt trên [Github](https://github.com/markmanx/isopacks).

Ngoài ra, các Isopacks cho **AWS**, **Azure**, **GCP**, và **Kubernetes** cũng được bao gồm.
Bạn có thể chọn những Isopack nào để nhập vào ứng dụng của mình và bỏ lại những cái nào.

### Tải Isopacks vào Isoflow

1. Cài đặt gói `npm`:

```bash
npm i @isoflow/isopacks
```

2. Nhập các Isopacks bạn đã chọn:

```jsx showLineNumbers
import Isoflow from 'isoflow';
import { flattenCollections } from '@isoflow/isopacks/dist/utils';
import isoflowIsopack from '@isoflow/isopacks/dist/isoflow';
import awsIsopack from '@isoflow/isopacks/dist/aws';
import gcpIsopack from '@isoflow/isopacks/dist/gcp';
import azureIsopack from '@isoflow/isopacks/dist/azure';
import kubernetesIsopack from '@isoflow/isopacks/dist/kubernetes';

const icons = flattenCollections([
  isoflowIsopack,
  awsIsopack,
  azureIsopack,
  gcpIsopack,
  kubernetesIsopack
]);

const App = () => {
  return (
    <Isoflow initialData={{
      title: 'Example scene',
      icons,
      colors: [],
      items: [],
      views: []
    }} />
  );
}

export default App;
```

## Sử dụng không cần Isoflow

Isopacks cũng có thể được sử dụng mà không cần Isoflow hoặc React (ví dụ: bạn có thể kéo thả trực tiếp hình ảnh vào slide hoặc tài liệu, hoặc nhập vào dự án Javascript / Typescript cơ bản của bạn).
Xem [dự án Github Isopacks](https://github.com/markmanx/isopacks) để biết thêm thông tin.

## Tự lưu trữ so với nhập biểu tượng

Mặc dù bạn có thể nhập trực tiếp hình ảnh biểu tượng vào ứng dụng JS hoặc TS của mình, nhưng chúng tôi khuyên bạn nên tự lưu trữ hình ảnh biểu tượng để chúng có thể được tải chậm (lazy-loaded) bằng cách tham chiếu chúng thông qua URL từ một dịch vụ như S3 hoặc CDN.
